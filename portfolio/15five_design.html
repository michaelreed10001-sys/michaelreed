15Five Design System

Building a scalable design system from scratch to unify product experiences, accelerate development, and maintain brand consistency across a growing platform.

My Role

Lead Product Designer responsible for strategy, research, component design, documentation, and implementation.

The Team

Product Manager, 4 Front-end Engineers, Marketing Team collaborating in bi-weekly sprints.

Timeline

4-month project from discovery to launch, with ongoing maintenance and component library expansion.

THE CHALLENGE

From Inconsistency to Unified Design

15Five's rapid growth led to fragmented UI patterns, inconsistent user experiences, and inefficient design-development workflows that hindered productivity and brand cohesion.

3

Hindered Productivity

Lack of standardized design elements slowed down both design and development processes significantly.

4

Communication gaps between design and engineering led to duplicated efforts and misaligned implementations.

5

Brand Identity Erosion

Inconsistent application of colors, typography, and visual elements diluted the 15Five brand experience.

Looking at Design System Methods

Before going into the details of the design system, I looked at the methodologies that would help structure the design system effectively.

Material Design

Developed by Google, it provides a set of guidelines and components for creating a consistent design across platforms. Offers comprehensive documentation and accessibility standards.

Brad Frost's Atomic Design

This method breaks down the design into basic building blocks like atoms, molecules, organisms, etc., to ensure consistency and scalability. Provides a clear mental model for component architecture.

Design Tokens

Abstracting design properties like colors, typography, and spacing into reusable tokens, which can be implemented across different platforms. Enables systematic theming and brand consistency.

InVision Design System Manager

A platform to create and manage design systems collaboratively, facilitating version control and updates. Provides workflow integration for design and development teams.

Hybrid Methodology Approach

Each methodology comes with its own set of advantages and drawbacks. To create our design system, I opted for a pragmatic approach, cherry-picking the most fitting elements from each method. This ensures that we incorporate the aspects that align best with the requirements and vision of our product.

We adopted Atomic Design's mental model for component architecture, Material Design's accessibility guidelines, Design Tokens for theming consistency, and collaborative workflows inspired by DSM platforms.

IMPLEMENTATION

Building the Foundation

We established weekly cross-functional check-ins to align design and engineering, ensuring collaborative development and practical implementation.

1

Color Foundation

In our initial check-in, I kicked off with a fundamental aspect of the design system - color. Together, we delved into the existing color palette, addressing essential questions and pinpointing various issues. We explored whether all colors were effectively utilized, identified specific use cases, and ensured alignment between the colors in Figma and the production environment.

Streamlining the palette by removing unnecessary colors and identifying expansion areas, I collaborated with the marketing team to align it with the 15Five brand. This review prompted the addition of a few extra brand colors for better semantic usage.

Color system showing semantic colors, brand alignment, and implementation specifications

2

Typography System

We conducted a thorough review of the existing typography system, identifying gaps in the type scale and inconsistencies between design and implementation. The comprehensive palette now details each core color component, its designated use, corresponding name in our front-end library, and the hex code.

Refining the typography aspect of the existing system, I introduced enhancements such as Body Emphasized and Body Small text to cater to captions and key phrases. Each font style now comes with applied line spacing and usage text, providing users with added context when incorporating the style into their designs.

Typography system with comprehensive type scale, usage guidelines, and implementation specs

3

Button Components

Buttons were inconsistently applied in the existing product, with some pages featuring up to 4 primary buttons, and variations in link buttons with or without icons. Moreover, the existing button component lacked scalability with text and lacked flexible variations for icons or menus.

This led to additional time spent on mockups and often resulted in detached components to adjust the button's appearance. Detached components pose a challenge to an efficiently managed design system, as they don't automatically update when changes are made to the primary component.

Comprehensive button system with primary, secondary, tertiary variants and all interactive states

4

Icon Library

Opting not to invest in building a custom icon library due to ROI considerations, I instead imported over 2,100 icons from Google's Material Design. In just 5 minutes, we established a fully searchable and accessible icon library, providing our front-end team with the same resource and achieving icon parity efficiently.

Now, when a designer toggles on the icon option for a button, they can search through an entire library of ready-to-use icons for their button! The icon library includes both filled and outlined variants, organized by categories for easy discovery.

Animated demonstration of the searchable icon library with 2,100+ Material Design icons in filled and outlined variants

Input Components

Many design systems I assessed overcomplicated input components, leading to an abundance of variants and states. The existing system lacked the benefits of auto-layout and component properties, prompting a fresh start.

Simplifying Complex Inputs

Utilizing nested master components for inputs, I ensured efficient updates to shared styles like the gray border and placeholder text. This eliminated the need to adjust these values for each component individually.

The result is a set of four distinct input components: text, search, select, and date pickers. Each component boasts toggle-able states for default, hover, focus, error, and disabled, streamlining the design process.

Error states seamlessly accommodate custom error text, wrapping neatly to the size of the input field. For specific components such as select or date pickers, a unique state is available to toggle an expandable menu that scales appropriately with the desired number of options.

Figma Design System Layout

The design system was implemented in Figma with a clear, organized structure that mirrors our component architecture and makes it easy for designers to find and use the right components.

The complete 15Five design system implemented in Figma with organized components and documentation

Organized for Efficiency and Clarity

The Figma file was structured into clear sections: Foundations (colors, typography, icons), Components (buttons, inputs, modals, etc.), Patterns (common UX patterns), and Utilities (annotation components, spacing helpers).

This organization allowed designers to quickly locate the components they needed while understanding the relationships between different elements. The use of consistent naming conventions and clear documentation within Figma itself helped reduce the learning curve and ensure consistent usage across the design team.

Storybook Input Components

Storybook was used to store comprehensive documentation about the design system, providing an interactive environment for developers to explore components, view code examples, and understand implementation details.

Interactive Documentation for Developers

Storybook served as the single source of truth for component documentation in the engineering workflow. Each component was documented with:

  • Live interactive examples that developers could manipulate to see different states and variations
  • Code snippets showing implementation examples in React
  • Props documentation with type definitions, default values, and usage guidelines
  • Accessibility considerations and keyboard navigation patterns
  • Design guidelines linking back to the Figma design system

This approach ensured that developers had all the information they needed to implement components correctly without constantly needing to consult designers. The interactive nature of Storybook also made it easier for new team members to onboard and understand the component library.

Key Learnings

This project reinforced critical principles for successful design system implementation:

Collaboration is Essential

Involving engineers from the start ensured practical components and full adoption. Weekly check-ins created alignment and shared ownership.

Simplicity Over Complexity

Streamlined systems with clear choices outperform complex systems with endless options. Reducing decision fatigue increases adoption.

Documentation Drives Adoption

Comprehensive documentation with real examples and clear guidelines became the single source of truth, accelerating onboarding and consistency.

Maintenance is Critical

The real challenge lies not in the initial creation but in ongoing maintenance. Failing to consider upkeep effort can result in the system being overlooked and neglected.
