PROJECT OVERVIEW

System Consolidation for More Than Insurance

The Challenge

More Than Insurance identified an excessive number of systems that employees had to access to complete customer insurance claims. The Policy system, developed by programmers without UX expertise, needed consolidation under a single user interface. This presented an opportunity to enhance both employee efficiency and customer satisfaction through improved claims processing.

Key Issues Identified

1

Multiple System Access

Employees had to access multiple systems and duplicate data entry to process a single claim, leading to inefficiency and errors.

2

Lack of System Overview

Due to system complexity, users couldn't get a clear overview of claims areas, making it difficult to manage workloads effectively.

3

Manual Processes

Too many manual processes that could be automated, reducing system efficiency and increasing processing time.

4

Poor User Experience

The programmer-developed system lacked UX expertise, resulting in confusing workflows and inefficient task completion.

The Goal

To create a unified Guidewire-based policy management system that consolidates multiple systems into a single interface, automates manual processes, and provides employees with comprehensive overviews and drill-down capabilities for efficient claims processing.

Competitor Analysis

To ensure our solution would meet industry standards and user expectations, we conducted a thorough analysis of competitor approaches to policy management and claims processing systems. We examined both direct insurance competitors and best-in-class enterprise software interfaces to identify patterns, strengths, and opportunities for differentiation in our design.

Analysis of competitor policy management interfaces and workflows

Comparative evaluation of claims processing user experiences

Key Insights from Competitor Analysis

Industry Standard Patterns: Most competitors used dashboard-centric designs with summary cards and drill-down capabilities, establishing user expectations we needed to meet.

Communication Gaps: Even advanced systems struggled with integrated communication hubs, presenting an opportunity for differentiation in our design.

Mobile Responsiveness: Few enterprise insurance systems offered true mobile optimization, suggesting an area where we could exceed expectations.

Guidewire Customization: Analysis of other Guidewire implementations revealed common customization patterns we could leverage while avoiding known usability pitfalls.

Research Findings

Our research revealed consistent pain points across both customer and employee experiences, highlighting the need for a unified system with better communication and automation capabilities.

Customer Findings

Lack of Claim Visibility: Customers required regular communication and updates about their claim status, which was not consistently provided.
Poor Supplier Perception: Customers had a negative image of supplier commitment to customer care during the claims process.
Inadequate Support: Customers felt there was a lack of reassurance and support throughout the claims process.
Inconvenient FNOL Process: The First Notification of Loss (FNOL) process was found to be inconvenient and time-consuming.
Inefficient Processes: Customers perceived existing processes as inefficient and unfair for handling claims.

Employee Findings

System Transparency Issues: More Than and partner organizations couldn't view each other's systems, with insufficient integration between platforms.
Multiple System Access: Employees had to access multiple systems for claims tasks, which was time-consuming and inefficient.
Limited System Functionality: Claims administration systems lacked necessary functionality, impacting productivity (e.g., small text boxes for notes).
Communication Gaps: Suppliers and brokers felt under-informed about claim progress, with some brokers unaware when clients made claims.

Key Findings from Prototype Testing

Dashboard Information Hierarchy: Initial designs presented too much information at once. We simplified the default view with expandable sections for detailed data.
Communication Workflow: Users wanted clearer differentiation between automated and manual communications. We added visual indicators and separate tabs for each type.
Supplier Portal Navigation: External partners found initial navigation confusing. We simplified the supplier portal with task-based navigation.
Mobile Responsiveness: While primarily a desktop system, users wanted mobile access for basic functions. We optimized key workflows for tablet use.
Search & Filter Functionality: Users needed more powerful search across all claim data. We enhanced search with filters for status, date, type, and assigned handler.
DESIGN REFINEMENTS

Iterating Based on User Feedback

Based on prototype testing findings and stakeholder feedback, we made several key improvements to the system design:

1

Simplified Dashboard

Redesigned the default dashboard view to show only critical information, with expandable sections for detailed data based on user role and task.

2

Enhanced Communication Hub

Separated automated and manual communications, added visual status indicators, and implemented templates for common messages.

3

Streamlined Supplier Portal

Redesigned the supplier interface with task-based navigation, clearer instructions, and simplified report submission.

4

Automated Workflow Triggers

Implemented intelligent automation that triggers workflows based on claim type, value, and complexity, reducing manual steps by 40%.

Regular communication with diverse stakeholders (employees, customers, suppliers, brokers) is essential for successful system transformation.
