PROJECT OVERVIEW

Project Saturn: Modernizing Insurance Market Intelligence

The Challenge

Gallagher's current quarterly reporting process for insurance carriers was entirely manual, PDF-based, and inefficient. Market insights were produced quarterly through interviews with Gallagher Re and specialty teams, then compiled into static PowerPoint and PDF documents. This process lacked automation, created version control issues, and provided a poor user experience for both internal teams and external clients like Allianz.

Key Issues Identified

1

Manual & Inefficient Process

Current reporting required manual data compilation from multiple sources (Gallagher Re, Gallagher Drive) with no automation, leading to time-consuming quarterly cycles.

2

Static PDF Limitations

Users had to navigate between Excel data and PDFs to access detailed information, with no interactivity or customization options.

3

Lack of Real-Time Insights

Brokers formed subjective market views that were manually integrated into reports, with no ability to provide instantaneous market data.

4

Poor User Experience

Clients couldn't customize views, filter data, or access historical trends easily. The process was confusing with unclear colored arrows and messaging.

The Goal

To create a white-labeled, interactive KPI reporting platform that replaces manual PDF reports with real-time data visualization, customizable dashboards, and streamlined workflow management for insurance carriers, starting with Allianz as the pilot client.

Research Findings

Our research revealed consistent patterns across stakeholder groups. Internal teams wanted efficiency gains and better collaboration tools, while external clients demanded more interactive, customizable, and timely market intelligence.

Key Requirements Identified

Dashboard & KPI Visualization: Users need summary KPIs for all Lines of Business (LOBs) with house views, PDF links, and performance KPIs showing quarter-on-quarter and year-on-year trends.
Quarter Lifecycle Management: The system needs Published and Draft statuses for quarterly data, with auto-creation of next quarters and clear progress indicators.
User Management & Access Control: Platform requires robust user management with admin capabilities, SSO integration, and restrictions to prevent multiple users editing the same LOB.
White-Labeling & Branding: Each carrier needs distinct landing pages with their logos and color schemes, with PDF downloads including respective branding.
Historical Data Access: Users want configurable year and quarter pickers for historical data, with version control for draft and published states.

Key Findings & Iterations

KPI Visualization Clarity: Users initially found some trend indicators confusing. We simplified the visual language and added tooltips explaining QoQ/YoY calculations and what colored arrows represent.
Filter Complexity: The initial filtering interface had too many options visible at once. We implemented progressive disclosure, showing basic filters by default with advanced options available on demand.
Admin Workflow Efficiency: Administrative tasks required too many clicks. We consolidated related actions, added batch operations, and implemented keyboard shortcuts for power users.
Mobile Responsiveness: While primarily a desktop platform, users wanted limited mobile access for quick checks. We optimized key views for tablet and mobile while focusing on desktop for complex tasks.
Terminology Consistency: Different user groups used varying terminology for the same concepts. We standardized on industry-standard terms with tooltips explaining Gallagher-specific terminology.
DESIGN REFINEMENTS

Iterating Based on User Feedback

Based on usability study findings and stakeholder feedback, we made several key improvements to the platform design:

1

Simplified KPI Visualization

Redesigned trend indicators with clearer visual language, consistent color coding, and explanatory tooltips to reduce confusion about QoQ/YoY metrics.

2

Enhanced Filtering Interface

Implemented progressive disclosure for filters, saving frequently used filter combinations, and clearer visual feedback for active filters.

3

Streamlined Admin Workflows

Consolidated administrative tasks, added bulk operations for user management, and implemented search functionality across all admin sections.

4

Improved Data Navigation

Enhanced breadcrumb navigation, added "back to dashboard" shortcuts, and implemented persistent filtering across related views.

Data Management & Version Control

A critical refinement was implementing robust data versioning and historical access controls based on stakeholder requirements:

Historical Data Access

Configurable year and quarter pickers with access controlled at contract level. Some clients don't subscribe to historical data.

Version Control

Draft and published states managed to avoid inconsistent reporting periods. Clear timeline for publishing data.

Archival Storage

Documentation archival with version history. Ability to access previous versions of reports for audit purposes.

RESULTS & LEARNINGS

Impact and Key Takeaways

What We Learned

The redesign process taught us valuable lessons about enterprise UX, data visualization, and stakeholder management in the insurance industry:

Enterprise UX Complexity

Designing for multiple user roles with different permissions requires careful consideration of information architecture and access controls.

Data Visualization Best Practices

Insurance professionals have specific expectations for how data should be presented. Industry-standard visualizations build trust faster than novel approaches.

White-Labeling Implementation

Creating a design system that supports easy rebranding while maintaining consistent UX patterns requires careful abstraction of styling and components.
